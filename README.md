# 🌐 Cloud/DevOps Roadmap — Gabriel Wietzikoski

Este repositório documenta minha jornada de estudos e projetos práticos em **Cloud Computing e DevOps**, alinhados com a Pós-Graduação em Redes de Computadores e IoT (UTFPR) e meu plano paralelo de 16 meses de evolução técnica.

---

## 📌 Objetivos
- Consolidar conhecimentos em **Linux, Redes, Cloud, DevOps e Observabilidade**.
- Construir um **portfólio público** com projetos reais e documentados.
- Demonstrar habilidades práticas que substituem certificações iniciais.
- Evoluir para atuar como **Cloud Engineer / DevOps Engineer**.

---

## 📂 Estrutura do Repositório
- `/Aulas` → resumos da pós-graduação.
- `/Linux` → Laboratórios e práticas de administração de sistemas.
- `/Redes` → Configurações, testes e análises com Wireshark e SNMP.
- `/AWS` → Projetos de infraestrutura em nuvem com AWS Free Tier.
- `/Docker` → Containers e aplicações dockerizadas.
- `/Kubernetes` → Manifests, clusters e observabilidade.
- `/Terraform` → Infraestrutura como código (IaC).
- `/Projetos` → Projetos completos integrando várias tecnologias.

---

## 🚀 Projetos Obrigatórios
1. **Infraestrutura AWS com Terraform**  
   - VPC, subnets, EC2, S3, Auto Scaling, Load Balancer.  
   - Documentação + diagrama de arquitetura.  

2. **Pipeline CI/CD completo**  
   - Aplicação simples, Docker, GitHub Actions/GitLab CI.  
   - Deploy automatizado em EC2 ou Kubernetes.  

3. **Cluster Kubernetes com Observabilidade**  
   - Deploy de aplicação web, Ingress + TLS.  
   - Prometheus + Grafana com dashboards customizados.  

4. **Monitoramento corporativo**  
   - SNMP, métricas de rede, alertas básicos.  

5. **IoT + Cloud (opcional)**  
   - Dispositivo coletando dados e enviando para AWS IoT Core.  
   - Visualização em Grafana.  

---

## 📅 Cronograma de Estudos
- **Meses 1–4:** Linux, Redes e AWS básico.  
- **Meses 5–8:** AWS avançado + Projeto de Infraestrutura.

- **Meses 13–16:** Segurança, Observabilidade e projetos finais.
